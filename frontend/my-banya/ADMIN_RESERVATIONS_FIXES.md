# Исправления раздела «Бронирования» (`/admin/reservations`)

Кратко зафиксированы правки по странице и связанным модалкам, чтобы было проще сопровождать код и не терять контекст.

## `AdminReservationsNew.jsx`

1. **Заголовок «Записи на …»** — дата для подписи строится через уже существующую локальную полуночь `parseLocalYmd(filters.date)`, а не через `new Date('YYYY-MM-DD')`, чтобы в разных часовых поясах не «прыгал» день.
2. **Табы «Все бани» / конкретная баня** — сравнение `activeBathId` и `bath.bath_id` приведено к строкам, чтобы не ломался выбор вкладки из‑за `number` vs `string` из `select` и API.
3. **Фильтр колонок** (`filteredBaths`) — то же строковое сравнение `bath_id`, чтобы список колонок совпадал с выбранной вкладкой.
4. **Удаление** — вызов API удаления игнорируется для искусственных записей уборки (`reservation_id` вида `cleaning-…`), чтобы случайно не дергать бэкенд с невалидным id.

## `AddBookingModal.jsx`

1. **Редактирование прошлых броней** — проверки «дата не в прошлом» и «время начала не в прошлом» применяются только при **создании** брони. При **редактировании** они отключены, иначе нельзя было сохранить изменения по уже прошедшим записям.
2. **Ранее в этой ветке (для справки)**:
   - однократная подстановка данных брони при открытии редактирования (без сброса добавленных товаров при догрузке склада);
   - поле «Количество гостей»: обычный текст, только цифры, можно очистить, ведущие нули убираются;
   - согласование id товаров через `Number(...)` при выборе из каталога.

## `ReservationsFilters.jsx`

1. **Двойной вызов `onApply`** — при смене даты в `onChange` больше не вызывается `onApply` напрямую; обновление родителя идёт через уже существующий `useEffect([date, onApply])`, чтобы не дублировать запросы/обновления состояния.

## `BookingDetailsModal.jsx`

1. **Импорты** — вспомогательная `formatLocalYmd` перенесена под блок импортов (корректный порядок для bundler/линтеров).
2. **Поиск бани** — `bath_id` сравнивается как строка с `booking.bath_id`, чтобы находить баню при расхождении типов.
3. **Сумма чека** — `total_cost` приводится к числу; при отсутствии/NaN итог и «баня только» не ломают рендер и не дают `toLocaleString` на `undefined`.
4. **Редактирование из карточки** — в `AddBookingModal` передаётся `selectedDate`, вычисленный из `booking.start_datetime` (локальный `YYYY-MM-DD`), чтобы вложенная форма опиралась на дату брони, а не на случайные дефолты.

## Заметки

- Полный прогон E2E по сценариям (создание, правка прошлой брони, смена даты в фильтре, табы бань) имеет смысл выполнить вручную в браузере после деплоя.
